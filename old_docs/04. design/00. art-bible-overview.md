# 아트바이블 개요

## 목적

이 아트바이블은 로그라이크 덱빌더 게임의 시각적 정체성을 정의하고, Gemini 3.0 API를 통한 일관된 아트 에셋 생성을 위한 가이드라인을 제공합니다.

---

## 아트 디렉션 철학

### 핵심 비전
> **"따뜻한 동화책 속 귀여운 모험"**

손으로 그린 듯한 따뜻한 스토리북 일러스트 스타일로, 귀여운 캐릭터들이 카드 형태로 모험하는 아기자기한 세계를 표현합니다.

### 시각적 목표
1. **친근함**: 귀여운 SD 비율의 캐릭터로 친근한 느낌
2. **따뜻함**: 손그림 느낌의 연필 스케치 + 수채화 채색
3. **가독성**: 심플한 프레임과 명확한 정보 표시
4. **일관성**: 모든 에셋이 하나의 동화책처럼 통일된 스타일

---

## 핵심 키워드

| 카테고리 | 키워드 |
|----------|--------|
| **스타일** | 손그림, 스토리북, 연필 스케치, 수채화 |
| **분위기** | 따뜻한, 아기자기한, 귀여운, 포근한 |
| **캐릭터** | SD 비율, 큰 머리, 볼 홍조, 따뜻한 표정 |
| **프레임** | 심플한 나무 프레임, 갈색, 금색 테두리 |

---

## 참조 게임

### 주요 참조: Dice & Fold
- **그림체**: 손으로 그린 스토리북 일러스트
- **캐릭터**: 과장된 비율, 큰 머리, 귀여운 느낌
- **색상**: 따뜻한 톤, 부드러운 색채
- **프레임**: 심플한 나무/갈색 프레임
- **분위기**: 아기자기하고 포근한 동화 느낌

### 게임플레이 참조: Slay the Spire
- **카드 UI**: 명확한 정보 계층, 즉각적 인지
- **전투 화면**: 적-아군 배치, 상태 표시

---

## 기술적 목표

### Gemini AI 생성 일관성
1. **마스터 프롬프트 시스템**: 모든 에셋에 적용되는 기본 스타일 레이어
2. **에셋별 템플릿**: 캐릭터/몬스터/배경/UI 각각의 전용 프롬프트
3. **네거티브 프롬프트**: 피해야 할 스타일 명시 (3D, 사실적, 어두운)
4. **검증 체크리스트**: 생성된 이미지의 스타일 일관성 확인

### 해상도 및 포맷
| 에셋 유형 | 권장 해상도 | 비율 |
|-----------|-------------|------|
| 캐릭터 카드 | 512 x 768 | 2:3 |
| 몬스터 카드 | 512 x 768 | 2:3 |
| 배경 | 1920 x 1080 | 16:9 |
| 아이콘 | 128 x 128 | 1:1 |
| 코인 | 256 x 256 | 1:1 |
