# UI 비주얼 가이드

## 개요

이 문서는 게임 UI의 시각적 규칙을 정의합니다. 모든 UI 요소는 스토리북 일러스트 스타일과 조화를 이루어야 합니다.

---

## UI 디자인 원칙

### 핵심 원칙
```
1. 가독성 우선: 게임플레이 정보는 즉시 인지 가능해야 함
2. 세계관 통합: UI도 동화책의 일부처럼 느껴져야 함
3. 비침습적: 게임 화면을 가리지 않고 보조적 역할
4. 일관성: 모든 UI 요소가 동일한 스타일 언어 사용
```

### 스타일 방향
- **손으로 그린 느낌**: 완벽한 직선보다 약간의 불완전함
- **양피지 텍스처**: UI 배경에 따뜻한 종이 질감
- **장식적 프레임**: 중세 필사본 스타일의 테두리
- **통합된 아이콘**: 이모지가 아닌 일러스트 스타일 아이콘

---

## UI 레이어 구조

### Canvas 계층
```
최상단: OverlayCanvas (z: 300)
├── 모달/팝업
├── 툴팁
└── 시스템 알림

중상단: HandCanvas (z: 200)
├── 손패 카드
└── 카드 드래그 영역

중단: BattleCanvas (z: 100)
├── 캐릭터 카드
├── 적 카드
├── 체력바/상태
└── 코인 시스템

하단: BGCanvas (z: 0)
├── 배경
└── 환경 효과
```

### 패널/카드 프레임
```
기본 프레임:
├── 배경: Parchment #F5E6D3
├── 테두리: Deep Sepia #3D2914, 4px
├── 모서리: 8px radius (약간 둥글게)
├── 그림자: 10px blur, 20% opacity
└── 내부 여백: 16px
```

## 애니메이션 규칙

### 기본 이징
```javascript
// 자연스러운 동작
easeOutQuad: 부드럽게 감속
easeInOutCubic: 강조 동작

// 탄성 효과
easeOutBack: 약간 넘어갔다 돌아오는 느낌
easeOutBounce: 통통 튀는 느낌 (보상 획득 등)
```

### 지속 시간 가이드
| 동작 유형 | 시간 | 용도 |
|-----------|------|------|
| 즉각 피드백 | 100ms | 버튼 클릭, 호버 |
| 상태 전환 | 200-300ms | 패널 열기/닫기 |
| 강조 효과 | 400-600ms | 데미지 표시, 획득 |
| 드라마틱 | 800-1200ms | 코인 뒤집기, 레벨업 |
