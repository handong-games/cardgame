# Unity 폴더 구조

> Unity 프로젝트의 Assets 폴더 구조 정의

---

## 폴더 구조

```
Assets/
├── @Editor/                     # 에디터 전용 스크립트
│
├── @Resources/                  # 게임 리소스 (우리가 만든 에셋)
│   ├── Prefabs/                 # 프리팹
│   │   ├── UI/                  # UI 프리팹
│   │   └── Battle/              # 전투 관련 프리팹
│   ├── Sprites/                 # 스프라이트
│   │   ├── Cards/               # 카드 아트
│   │   ├── Characters/          # 캐릭터 이미지
│   │   └── UI/                  # UI 이미지
│   ├── Audio/                   # 오디오
│   │   ├── BGM/
│   │   └── SFX/
│   └── Fonts/                   # 폰트
│
├── @Scripts/                    # C# 스크립트
│   ├── Core/                    # 핵심 시스템 (싱글턴, 유틸리티 등)
│   ├── ScriptableObjects/       # SO 정의 클래스
│   │   ├── CardData.cs
│   │   ├── EnemyData.cs
│   │   ├── BuffData.cs
│   │   └── RegionData.cs
│   ├── Features/                # 기능별 (하이브리드 구조)
│   │   ├── Battle/              # 전투 로직
│   │   ├── Card/                # 카드 시스템
│   │   └── Meta/                # 메타 시스템 (런, 보상 등)
│   ├── Managers/                # 매니저 클래스
│   └── UI/                      # UI 컴포넌트
│
├── @ThirdParty/                 # Asset Store 에셋, 외부 SDK
│   ├── DOTween/                 # (예시)
│   ├── Spine/                   # (예시)
│   └── Odin/                    # (예시)
│
├── Plugins/                     # 네이티브 DLL/플랫폼 종속만
│   └── Steamworks.NET/          # 스팀 SDK
│
├── Scenes/                      # 씬 파일 (Unity 기본 위치)
│   ├── BootScene.unity
│   └── BattleScene.unity
│
├── TextMeshPro/                 # Unity 패키지 자동 생성 (유지)
├── AddressableAssetsData/       # Addressables 자동 생성 (유지)
└── MCPForUnity/                 # MCP 플러그인 (개발용)
```

---

## 폴더 분류 원칙

| 폴더 | 용도 | 예시 |
|------|------|------|
| **@Editor** | 에디터 전용 스크립트 | Custom Inspector, 에디터 툴 |
| **@Resources** | 우리가 만든 게임 에셋 | Prefabs, Sprites, Audio, Fonts |
| **@Scripts** | C# 스크립트 | 모든 런타임 코드 |
| **@ThirdParty** | Asset Store/외부 SDK | DOTween, Spine, Odin |
| **Plugins** | 네이티브 DLL만 | Steamworks.NET DLL |
| **Packages (UPM)** | 공식/오픈소스 패키지 | Addressables, Newtonsoft, TextMeshPro |
| **Scenes** | 씬 파일 | Unity 기본 위치 유지 |
| **Unity 자동생성** | 패키지 필수 자산 | TextMeshPro/, AddressableAssetsData/ |

---

## @ 접두사 사용 이유

- Assets 폴더 내에서 **정렬 최상단** 유지
- 프로젝트 코드/에셋과 **외부 에셋 구분** 명확
- 한눈에 "우리 코드"와 "외부 코드" 식별 가능

---

## 외부 패키지 분류 기준

### @ThirdParty (Asset Store / 외부 SDK)
- Asset Store에서 구매한 에셋
- Unity 공식 패키지가 **아닌** 외부 SDK
- UPM 미지원 라이브러리

**예시:**
- DOTween (Asset Store 버전)
- Spine-Unity
- Odin Inspector
- 외부 CSV 파서

### Plugins (네이티브 DLL)
- 플랫폼 종속 바이너리
- 네이티브 DLL 파일

**예시:**
- Steamworks.NET (steam_api64.dll 등)
- 플랫폼별 네이티브 플러그인

### Packages (UPM)
- Unity Package Manager로 관리
- `manifest.json`에서 버전 고정

**예시:**
- com.unity.addressables
- com.unity.textmeshpro
- com.unity.nuget.newtonsoft-json
- DOTween (UPM 버전, 지원 시)

## @Scripts 하위 구조 (하이브리드)

```
@Scripts/
├── Core/                    # 핵심 시스템
│   ├── Singleton.cs
│   ├── ServiceLocator.cs
│   └── Utilities/
│
├── ScriptableObjects/       # SO 정의 클래스
│   ├── CardData.cs
│   ├── EnemyData.cs
│   ├── BuffData.cs
│   └── RegionData.cs
│
├── Features/                # 기능별 분류
│   ├── Battle/              # 전투 시스템
│   │   ├── BattleState.cs
│   │   ├── TurnSystem.cs
│   │   └── DamageCalculator.cs
│   ├── Card/                # 카드 시스템
│   │   ├── CardInstance.cs
│   │   ├── CardEffectResolver.cs
│   │   └── DeckController.cs
│   └── Meta/                # 메타 시스템
│       ├── RunState.cs
│       ├── RewardSystem.cs
│       └── ClassAdvancement.cs
│
├── Managers/                # 매니저 클래스
│   ├── GameManager.cs
│   ├── BattleManager.cs
│   ├── DeckManager.cs
│   └── UIManager.cs
│
└── UI/                      # UI 컴포넌트
    ├── Battle/
    │   ├── CharacterCardUI.cs
    │   ├── HPBarUI.cs
    │   └── ShieldIconUI.cs
    ├── Card/
    │   ├── HandCardUI.cs
    │   └── CardDragHandler.cs
    └── Common/
        ├── EnergyDisplayUI.cs
        └── EndTurnButtonUI.cs
```
