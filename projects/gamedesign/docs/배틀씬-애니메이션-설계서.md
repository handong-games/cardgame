# 배틀씬 애니메이션 설계서

> **기준 해상도**: 1920 × 1080px
> **애니메이션 엔진**: Framer Motion (React)
> **최종 수정일**: 2026-03-04
> **근거 소스**: `dev/proto-design/src/animations/`, `dev/proto-design/src/components/`

---

## 1. 개요

### 1.1 설계 원칙

1. **게임 속도 스케일링**: 모든 전투 애니메이션은 `getCurrentSpeedMultiplier()` 기반으로 속도 배수 적용
2. **물리 기반 스프링**: 카드/UI 요소는 `spring` 트랜지션 (stiffness/damping) 사용으로 자연스러운 반응
3. **시차 배치 (Stagger)**: 복수 요소 등장 시 index 기반 딜레이로 연속 등장감
4. **3단계 피드백**: 공격 → 피격 → 결과 순서의 전투 흐름 보장

### 1.2 애니메이션 라이브러리 구조

```
src/animations/
├── cardAnimations.ts      — 카드 드로우/버림 variants (Framer Motion)
├── coinAnimations.ts      — 동전 토스/결과 이펙트
├── combatAnimations.ts    — 전투 공격/방어/피격
└── index.ts               — 모듈 통합 내보내기

src/components/effects/
├── CoinTossAnimation.tsx  — 동전 비행/착지/이동 이펙트
├── GoldDrop.tsx           — 골드 획득 드롭 이펙트
└── SoulDrop.tsx           — 영혼 획득 아크 이펙트
```

### 1.3 속도 배수 시스템

| 설정 | 배수 | 적용 범위 |
|:-----|:-----|:---------|
| 느리게 | 1.5× | 모든 전투 애니메이션 |
| 보통 | 1.0× | 기본값 |
| 빠르게 | 0.5× | 모든 전투 애니메이션 |

```typescript
function getScaledCombatTiming() {
  const m = getCurrentSpeedMultiplier();
  return {
    PEEK_DURATION: 0.1 * m,
    HIT_DURATION: 0.15 * m,
    RETURN_DURATION: 0.15 * m,
    SHAKE_DURATION: 0.3 * m,
    SHIELD_SHAKE: 0.2 * m,
  };
}
```

## 2. 전투 애니메이션 (Combat Animations)

### 2.1 플레이어 공격

| 속성 | 값 |
|:-----|:---|
| 트리거 | 플레이어 스킬 사용 시 `isAttacking = true` |
| 속성 | x축 이동 |
| 키프레임 | `[0, -20, 30, 0]` (뒤로 → 전진 → 복귀) |
| 총 지속시간 | 0.4초 (PEEK 0.1 + HIT 0.15 + RETURN 0.15) |
| times | `[0, 0.25, 0.6, 1]` |
| easing | `easeOut` |
| 속도 배수 | ✅ 적용 |

**동작 설명**:
1. **0~25%**: x: 0→-20px (뒤로 준비 동작)
2. **25~60%**: x: -20→30px (전방 돌진)
3. **60~100%**: x: 30→0px (원위치 복귀)

### 2.2 적 공격

| 속성 | 값 |
|:-----|:---|
| 트리거 | 몬스터 페이즈에서 적 행동 실행 시 |
| 키프레임 | `[0, 20, -30, 0]` (뒤로 → 전진 → 복귀, 좌우 반전) |
| 총 지속시간 | 0.4초 |
| times | `[0, 0.25, 0.6, 1]` |
| easing | `easeOut` |
| 속도 배수 | ✅ 적용 |

### 2.3 피격 반응 (Hit React)

| 속성 | 값 |
|:-----|:---|
| 트리거 | `isHit = true` + 공격자의 PEEK+HIT 완료 후 |
| 딜레이 | (PEEK_DURATION + HIT_DURATION) × 1000 ms |
| 속성 | x축 진동 |
| 키프레임 | `[0, -10, 10, -5, 0]` |
| 지속시간 | 0.3초 (SHAKE_DURATION) |
| times | `[0, 0.2, 0.5, 0.8, 1]` |
| easing | `easeOut` |

**피격 틴트 오버레이**:

| 속성 | 값 |
|:-----|:---|
| 컴포넌트 | CharacterCard / EnemyCard 내 `motion.div` |

| 키프레임 | opacity: `[0, 0.6, 0]` |
| 지속시간 | 0.4초 |

## 3. 동전 토스 애니메이션 (Coin Toss)

### 3.1 타이밍 상수

| 상수 | 값 | 설명 |
|:-----|:---|:-----|
| TOSS_DURATION | 0.8초 | 기본 토스 모션 지속시간 |
| FLIGHT_DURATION | 0.6초 | 물리 기반 비행 지속시간 |
| COIN_STAGGER | 0.1초 | 동전 간 시차 딜레이 |
| LANDING_DISPLAY | 0.5초 | 착지 후 표시 시간 |
| TOTAL_DURATION | 1.1초 | 총 시퀀스 시간 |

### 3.2 동전 비행 (Flight Phase)

| 속성 | 값 |
|:-----|:---|
| 트리거 | 코인 주머니 클릭 시 |
| 시작점 | 코인 주머니 위치 (pouchPosition) |
| 궤적 | 포물선 (startX → peak → targetX) |
| 초기 상태 | opacity: 0, scale: 0.3, rotateY: 0, rotateX: 0 |

**키프레임**:

| 속성 | 0% | 40% | 100% |
|:-----|:---|:-----|:-----|
| x | startX | startX + 30% | targetX |
| y | startY | peakY (위쪽) | targetY |
| scale | 0.3 | 1.2 | 1 |
| rotateY | 0° | 360° | 랜덤 (최대 1080°) |
| rotateX | 0° | 180° | 360° |
| opacity | 0 | 1 | 1 |

| 속성 | 값 |
|:-----|:---|
| 지속시간 | FLIGHT_DURATION × 1.2 × speedMultiplier |
| 딜레이 | index × COIN_STAGGER × speedMultiplier |
| easing | cubic-bezier `[0.2, 0.8, 0.4, 1]` |
| 동전 크기 | 비행 중 w-10 h-10 (40×40px) |

**착지 궤적 생성 (generateCoinTrajectory)**:
- 산포 각도: baseAngle(-π/2) ± π/6 랜덤
- 비행 거리: 150~250px 랜덤
- targetY: startY - 80 ± 30px
- 여백 제한: 32px 마진

### 3.3 착지 대기 (Landing Phase)

| 속성 | 값 |
|:-----|:---|
| 지속시간 | 200ms × speedMultiplier |
| 상태 | `phase: 'landed'` → 대기 후 `phase: 'moving'` |

### 3.4 카운터 이동 (Moving Phase)

| 속성 | 값 |
|:-----|:---|
| 트리거 | 착지 대기 완료 후 자동 |
| 시작 | 착지 위치 (currentX, currentY) |
| 목표 | 앞면 → sunCountPosition / 뒷면 → moonCountPosition |
| scale | 1 → 0.5 |
| opacity | 1 → 0.8 |
| 지속시간 | 0.4초 × speedMultiplier |
| easing | cubic-bezier `[0.4, 0, 0.2, 1]` |
| 동전 크기 | 이동 중 w-8 h-8 (32×32px) |

### 3.5 결과 이펙트

**앞면 (태양) 이펙트**:

| 속성 | 값 |
|:-----|:---|
| boxShadow | `[0 0 0px, 0 0 20px #EAB308CC, 0 0 10px #EAB30880]` |
| scale | `[1, 1.1, 1]` |
| 지속시간 | 0.3초 |
| 딜레이 | 토스 완료 후 (TOSS_DURATION) |
| 글로우 색상 | `rgba(255, 215, 0, 0.6)` |

**뒷면 (달) 이펙트**:

| 속성 | 값 |
|:-----|:---|
| opacity | `[1, 0.5]` |
| scale | `[1, 0.9]` |
| 지속시간 | 0.2초 |
| 글로우 색상 | `rgba(192, 192, 192, 0.4)` |

### 3.6 전체 시퀀스 타임라인 (동전 5개 기준)

```
t=0ms        코인 0 비행 시작
t=100ms      코인 1 비행 시작
t=200ms      코인 2 비행 시작
t=300ms      코인 3 비행 시작
t=400ms      코인 4 비행 시작
t=720ms      코인 0 착지 (0.6×1.2=0.72초)
t=820ms      코인 1 착지
t=920ms      코인 0 → 카운터 이동 시작 (착지+200ms 대기)
...
t=1320ms     코인 0 카운터 도착 → onCoinLand 콜백
...
t=~1800ms    모든 코인 완료 → 100ms 후 onComplete
```
