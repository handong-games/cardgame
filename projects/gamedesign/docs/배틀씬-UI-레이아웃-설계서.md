# 배틀씬 UI 레이아웃 설계서

> **기준 해상도**: 1920 × 1080px
> **비주얼 스타일**: v4.0 Dark Frame Edition
> **최종 수정일**: 2026-02-23
> **근거 문서**: 전투-시스템-종합-기획서, 몬스터-행동-표시-시스템, 스킬-프리뷰-시스템-기획서, 라운드-진행바-기획서

---

## 1. 개요

### 1.1 설계 원칙

1. **정보 계층**: 생존 정보(HP, 방어도) > 전략 정보(코인, 스킬, 몬스터 의도) > 부가 정보(라운드, 소울)
2. **시선 흐름**: 좌하단(자신) → 우측(적) → 하단(스킬) → 우하단(턴 종료) — Z자 패턴
3. **동적 스케일링**: 1920×1080 기준 설계 후, `scale()` 변환으로 모든 해상도 대응
4. **최소 가독성**: 모든 텍스트 12px 이상, 모든 인터랙티브 요소 44×44px 이상 (터치 타겟)

### 1.2 전체 존 구조

```
┌──────────────────────────────────────────────────────────────┐ y=0
│  Zone A: 상단 HUD                                     h=60   │
├──────────────────────────────────────────────────────────────┤ y=60
│                                                              │
│                                                              │
│  Zone B: 전투 무대                                    h=740   │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤ y=800
│                                                              │
│  Zone C: 액션 바                                      h=280   │
│                                                              │
└──────────────────────────────────────────────────────────────┘ y=1080
```

| 존 | 위치 | 크기 | 화면 비율 | 역할 |
|:---|:-----|:-----|:---------|:-----|
| **Zone A** | x:0 y:0 | 1920 × 60 | 5.6% | 진행 상황, 자원 표시 |
| **Zone B** | x:0 y:60 | 1920 × 740 | 68.5% | 전투 캐릭터, 코인, 이펙트 |
| **Zone C** | x:0 y:800 | 1920 × 280 | 25.9% | 스킬 조작, 턴 관리 |

### 3.5 B-3: 전투 중앙 영역 (코인 조작 패널 포함)

전투 이펙트, 코인 조작이 집중되는 공간입니다. **코인주머니, 코인 현황, 턴 종료 버튼**이 수직으로 그룹화되어 코인 플립 영역과 통합됩니다. 턴 상태 pill은 Zone A 중앙(A-3)에 배치되며, 라운드 진행바는 Zone B 상단(B-6)에 배치됩니다.

| 속성 | 값 |
|:-----|:---|
| 위치 | x:380 y:0 (Zone B 내) |
| 크기 | 720 × 740px |

## 6. 동적 크기 조절 (반응형) 규칙

### 6.1 기본 스케일링 전략

**핵심 원칙**: 1920×1080을 기준 해상도로 고정 설계하고, CSS `transform: scale()`로 전체 스케일링합니다.

```typescript
// 스케일 계산
const BASE_WIDTH = 1920;
const BASE_HEIGHT = 1080;

function calculateScale(): number {
  const scaleX = window.innerWidth / BASE_WIDTH;
  const scaleY = window.innerHeight / BASE_HEIGHT;
  return Math.min(scaleX, scaleY); // 종횡비 유지, 작은 쪽에 맞춤
}

// 적용
const scale = calculateScale();
container.style.transform = `scale(${scale})`;
container.style.transformOrigin = 'top left';
```

### 6.2 해상도별 스케일 테이블

| 해상도 | 스케일 | 비고 |
|:------|:------|:-----|
| 2560 × 1440 | 1.333 | 4K 모니터, 선명도 유지 |
| 1920 × 1080 | 1.000 | 기준 (100%) |
| 1600 × 900 | 0.833 | — |
| 1366 × 768 | 0.711 | 노트북 일반, 최소 가독성 검증 필요 |
| 1280 × 720 | 0.667 | HD, 최소 지원 해상도 |

### 6.3 컴포넌트별 스케일링 규칙

| 컴포넌트 | 스케일링 방식 | 최소 크기 (절대값) |
|:---------|:------------|:----------------|
| Zone A (상단 HUD) | 전체 scale | 높이 40px 이상 |
| Zone B (전투 무대) | 전체 scale | — |
| Zone C (액션 바) | 전체 scale | 높이 200px 이상 |
| 스킬 카드 | 전체 scale | 100 × 140px 이상 |
| HP 바 텍스트 | 전체 scale + 최소 폰트 보장 | 10px 이상 |
| 상태 아이콘 | 전체 scale | 20 × 20px 이상 |
| 버튼 | 전체 scale | 44 × 44px 터치 타겟 |
| 툴팁 | 전체 scale | 200px 너비 이상 |

### 6.4 레터박스 처리

화면 종횡비가 16:9가 아닌 경우:

```typescript
// 16:9보다 넓은 화면 (21:9 울트라와이드 등)
// → 좌우 레터박스 (검정 띠)

// 16:9보다 좁은 화면 (4:3 등)
// → 상하 레터박스 (검정 띠)

const aspectRatio = window.innerWidth / window.innerHeight;
const targetRatio = 16 / 9;

if (aspectRatio > targetRatio) {
  // 좌우 레터박스
  const scaledWidth = window.innerHeight * targetRatio;
  const offsetX = (window.innerWidth - scaledWidth) / 2;
  container.style.marginLeft = `${offsetX}px`;
} else {
  // 상하 레터박스
  const scaledHeight = window.innerWidth / targetRatio;
  const offsetY = (window.innerHeight - scaledHeight) / 2;
  container.style.marginTop = `${offsetY}px`;
}
```

### 6.5 창 리사이즈 대응

```typescript
// 리사이즈 이벤트 핸들러
window.addEventListener('resize', () => {
  requestAnimationFrame(() => {
    const scale = calculateScale();
    container.style.transform = `scale(${scale})`;
    updateLetterbox();
  });
});

// debounce는 적용하지 않음 — requestAnimationFrame으로 충분
```
