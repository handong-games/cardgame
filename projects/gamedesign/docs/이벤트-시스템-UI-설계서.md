# 이벤트 시스템 UI 설계서

> **기준 해상도**: 1920 × 1080px
> **비주얼 스타일**: v4.0 Dark Frame Edition
> **최종 수정일**: 2026-02-26
> **근거 문서**: 이벤트-시스템-기획서, 경로-선택-시스템-기획서, 배틀씬-UI-레이아웃-설계서

---

## 1. 개요

### 1.1 설계 원칙

1. **호기심 보존**: 경로 선택 시 이벤트 카드는 내용을 숨기고, 진입 후에만 유형·선택지를 공개
2. **비파괴적 오버레이**: 전투 배경 위에 팝업, Zone A(상단 HUD)와 Zone C(스킬 슬롯)는 그대로 노출
3. **정보 투명성**: 선택지 코스트, 확률, 리스크가 한눈에 파악 가능 — 불확실성은 이벤트 선택 시점까지만 유지
4. **최소 가독성**: 모든 텍스트 12px 이상, 모든 인터랙티브 요소 44×44px 이상 (터치 타겟)

### 1.2 역할 경계 정의 (3축)

| 축 | 담당 | 자원 | 획득 가능 물품 | 핵심 메커니즘 |
|:---|:-----|:-----|:-------------|:------------|
| **경제(Economy)** | 상점 | 소울(◆) | 스킬, 전리품, 스킬 슬롯 | 명시적 가격, 완전 정보 |
| **기회비용(Opportunity Cost)** | 마을 | HP, 턴(시간) | 동료, 휴식(HP 회복), 퀘스트 | 무형 코스트, 선택의 무게 |
| **운(Luck)** | 이벤트 | 리스크 | 코인 성장, 스킬 변환, 유니크 보상 | 확률 기반, 고위험 고수익 |

**겹침 방지 규칙**:

- 이벤트에서 상점 전용 스킬 직접 판매 **금지** (상점 영역)
- 이벤트에서 동료 획득 **금지** (마을 영역)
- 이벤트 보상은 상점/마을에서 구할 수 없는 **유니크 효과 포함 필수**

### 1.3 이벤트 유형 4종

| 유형 | HP 리스크 | 포기 가능 | 출현 비율 | 악센트 색상 | 설명 |
|:----:|:--------:|:--------:|:--------:|:----------|:-----|
| **(A) 관대** | 없음 | 가능 | 40~50% | `#6B9E78` | 안전한 소소한 보상 |
| **(B) 의외** | 없음 | 불가 | 15~20% | `#8B7BB5` | 강제지만 전부 이득 |
| **(C) 유혹** | 있음 | 가능 | 25~30% | `#D4985A` | 리스크-리워드 선택 |
| **(D) 시련** | 있음 | 불가 | ~10% | `#C45555` | 강제 리스크 + 반드시 보상 |

### 1.4 진입 조건

| 속성 | 값 |
|:-----|:---|
| 진입 경로 | 경로 선택 화면에서 이벤트 카드(❓) 선택 |
| 전투 여부 | 없음 — 즉시 이벤트 오버레이 표시 |
| 배경 | 현재 지역의 전투 배경 이미지 유지 |
| 퇴장 | 선택 완료 후 결과 화면 → 계속 버튼 → 경로 선택 화면 복귀 |
| 포기 | (A), (C) 유형만: 포기 버튼으로 아무 효과 없이 즉시 퇴장 |

## 2. 경로 선택 화면의 이벤트 카드

### 2.1 카드 외관 ASCII

이벤트 카드는 **경험 여부와 관계없이 항상 동일한 외관**으로 표시됩니다.

```
┌────────────────┐
│                  │
│   [❓ 아이콘]    │  ← 64×64px, 중앙
│                  │
│    이벤트        │  ← 14px Bold, #FFF5E6
│                  │
│   내용:  ???     │  ← 12px, #FFF5E6 50%
│                  │
└────────────────┘
```

| 속성 | 값 |
|:-----|:---|
| 카드 크기 | 경로 선택 카드와 동일 (기획서 참조) |
| 아이콘 | ❓ 물음표, 64×64px, `#D4A574` 60% 불투명도 |
| 이름 | "이벤트" (고정), 14px Bold, `#FFF5E6` |
| 미리보기 | "???" 고정, 12px, `#FFF5E6` 50% |
| 배경 | `#1E1E24` → `#2A2A32` 그라데이션 |
| 테두리 | `#4A4A55` 2px, border-radius 8px |

### 2.2 상태별 스타일

| 상태 | 불투명도 | 테두리 | 추가 효과 |
|:-----|:--------|:------|:---------|
| 기본 | 100% | `#4A4A55` 2px | 없음 |
| 호버 | 100% | `#D4A574` 2px | y -8px 상승, `drop-shadow(0 4px 12px rgba(212,165,116,0.3))` |
| 선택 불가 | 50% | `#4A4A55` 1px | grayscale 60% |

## 3. 이벤트 오버레이 전체 레이아웃

### 3.1 레이어 구조

이벤트는 **전체 화면 오버레이** (z-index: 10, 모달 레벨)로 표시됩니다.

```
기존 레이어 (유지):
┌─ z:0   배경 이미지
├─ z:4   Zone A (상단 HUD) ← 유지 (소울 카운터 실시간 반영)
├─ z:4   Zone C (스킬 슬롯) ← 유지 (현재 장착 스킬 참조용)

이벤트 오버레이:
├─ z:10  어둡기 오버레이 (#000000 50%)
├─ z:11  이벤트 패널 (960 × 820px, 화면 중앙)
├─ z:12  선택지 카드 호버/프리뷰
├─ z:13  확인 팝업 (선택 확인)
└─ z:14  확률 연출 오버레이 (코인 플립 등)
```

## 6. E-3: 선택지 카드 영역

### 6.1 전체 레이아웃

| 속성 | 값 |
|:-----|:---|
| 위치 | E-2 하단 16px |
| 크기 | 896 × 자동높이 (2장: 172px, 3장: 264px) |
| 정렬 | 수평 중앙 |

선택지 카드 2~3장 수직 배치:

| 속성 | 값 |
|:-----|:---|
| 카드 크기 | 800 × 80px (가로형) |
| 카드 간격 | 12px |
| 전체 높이 | 2장: 2 × 80 + 1 × 12 = 172px / 3장: 3 × 80 + 2 × 12 = 264px |
| 수평 오프셋 | (896 - 800) / 2 = 48px (영역 좌측에서) |

### 6.2 개별 선택지 카드 구성 ASCII

```
┌───┬──────┬─────────────────────────────────────────┬────────────┐
│4px│ icon │ 선택지 텍스트                              │ 코스트/확률  │
│ ▌ │ 48²  │ "소울 8을 주고 물약 구매 (HP 8~12 회복)"   │  ◆8        │
│   │      │                                           │            │
└───┴──────┴─────────────────────────────────────────┴────────────┘
```

| 요소 | 위치 (카드 내) | 크기 | 스타일 |
|:-----|:-------------|:-----|:------|
| 악센트바 | 좌측 x:0 y:0 | 4 × 80px | 유형별 악센트 색상, border-radius 좌측만 8px |
| 아이콘 | x:20 y:16 | 48 × 48px | 선택지별 아이콘, `#2A2A32` 원형 배경 |
| 텍스트 | x:84 y:16 | 자동 × 24px | 16px Bold, `#FFF5E6` |
| 부가 설명 | x:84 y:44 | 자동 × 20px | 13px, `#FFF5E6` 60% |
| 코스트/확률 | 우측 x:-24 y:중앙 | 자동 × 자동 | 우측 정렬 |
| 카드 배경 | 전체 | 800 × 80px | `#2A2A32`, border-radius 8px |
| 카드 테두리 | 전체 | — | `#4A4A55` 1px |

### 6.3 선택지 카드 상태별 스타일

| 상태 | 배경 | 테두리 | 악센트바 | 추가 효과 |
|:-----|:-----|:------|:--------|:---------|
| 기본 | `#2A2A32` | `#4A4A55` 1px | 유형 색상 100% | 없음 |
| 호버 | 유형 악센트 8% 블렌드 | 유형 악센트 40% 1px | 유형 색상 100% | x +4px 이동, `drop-shadow(4px 0 12px rgba(악센트,0.2))` |
| 클릭 | 유형 악센트 12% 블렌드 | 유형 악센트 60% 2px | 유형 색상 100% | 밝기 -5%, 0.1s |
| 비활성 (자원 부족) | `#2A2A32` 60% | `#4A4A55` 1px | `#5A5F6B` | grayscale 40%, 코스트 텍스트 `#C45555` |

### 6.4 확률 표시 (코스트/확률 영역)

확률이 있는 선택지 (유형 C 이벤트)에는 수치와 프로그레스바를 표시합니다.

| 속성 | 값 |
|:-----|:---|
| 위치 | 카드 우측 영역 (x:-140, y:중앙) |
| 크기 | 120 × 48px |
| 수치 텍스트 | "70% 성공", 14px Bold, `#D4A574` |

## 10. 이벤트 경제 맥락

### 10.1 기회비용 기준

이벤트를 선택하면 일반 몬스터 전투를 포기합니다. 보상은 이 기회비용을 기준으로 설계됩니다.

```
일반 몬스터 전투 가치:
  + 소울 드롭: 8 소울
  + 스킬 드롭: 전투 승리 시 3개 중 1개 선택 기회
  + 몬스터 경험: 해당 몬스터 정보 공개
  - HP 소모: ~8~12 HP

이벤트 가치:
  + HP 미소모 (전투를 건너뛰므로) ≈ 8~12 HP 절약
  + 이벤트 보상 (유형에 따라 변동)
  ± HP 변동 (유형에 따라)
```

**원칙**: 이벤트의 순수 자원 기대값 < 전투의 자원 기대값. 단, 전투 대체 불가 고유 보상 포함.

### 10.2 HP 상태에 따른 선택 변화

| HP 상태 | 이벤트 선택 합리성 | 근거 |
|:--------|:----------------|:-----|
| HP 높음 (>50%) | 낮음 | 전투의 소울/스킬 가치 > HP 절약 |
| HP 중간 (25~50%) | 상황적 | 고유 보상 필요 여부에 따라 판단 |
| HP 낮음 (<25%) | 높음 | HP 절약 가치 > 소울/스킬, 단 (D) 시련 리스크 존재 |

### 10.3 유형별 기대값 가이드라인

| 유형 | 자원 기대값 | HP 리스크 | 고유 보상 | 총 가치 판단 |
|:-----|:----------|:---------|:---------|:-----------|
| (A) 관대 | 소울 3~8 상당 | 0 | 낮음 (정보, 소량 회복) | 전투 < 이벤트 (HP 절약 시) |
| (B) 의외 | 소울 8~12 상당 | 0 | 중간 (임시 스킬) | 전투 ≈ 이벤트 |
| (C) 유혹 | 기대값 기준 소울 5~15 | HP 5~10 | 높음 (코인 성장) | 변동폭 큼 |
| (D) 시련 | 소울 10+ 상당 | HP 5~8 | 매우 높음 (스킬 업그레이드) | 전투 < 이벤트 (고유 보상 때문) |
