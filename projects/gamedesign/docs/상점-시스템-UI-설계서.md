# 상점 시스템 UI 설계서

> **기준 해상도**: 1920 × 1080px
> **비주얼 스타일**: v4.0 Dark Frame Edition
> **최종 수정일**: 2026-02-24
> **근거 문서**: 상점-시스템-기획서, 소울-경제-기획서, 전리품-시스템-기획서, 마을-시스템-기획서

---

## 1. 개요

### 1.1 설계 원칙

1. **역할 분리**: 상점(경제) vs 마을(기회비용) vs 이벤트(운) — 3축 완전 분리
2. **비파괴적 오버레이**: 전투 배경 위에 팝업, Zone A(상단 HUD)와 Zone C(스킬 슬롯)는 그대로 노출
3. **정보 투명성**: 가격, 효과, 보유 소울이 한눈에 파악 가능
4. **최소 가독성**: 모든 텍스트 12px 이상, 모든 인터랙티브 요소 44×44px 이상 (터치 타겟)

### 1.2 역할 경계 정의 (3축)

| 축 | 담당 | 자원 | 획득 가능 물품 | 핵심 메커니즘 |
|:---|:-----|:-----|:-------------|:------------|
| **경제(Economy)** | 상점 | 소울(◆) | 스킬, 전리품, 스킬 슬롯 | 명시적 가격, 완전 정보 |
| **기회비용(Opportunity Cost)** | 마을 | HP, 턴(시간) | 동료, 휴식(HP 회복), 퀘스트 | 무형 코스트, 선택의 무게 |
| **운(Luck)** | 이벤트 | 리스크 | 코인 성장, 스킬 변환, 유니크 보상 | 확률 기반, 고위험 고수익 |

**겹침 방지 규칙**:

- 상점에서 HP 회복 판매 **금지** (마을 영역)
- 마을에서 소울 거래 **금지** (상점 영역)
- 이벤트 보상은 상점/마을에서 구할 수 없는 **유니크 효과만**

### 1.3 상점 진입 조건

| 속성 | 값 |
|:-----|:---|
| 진입 경로 | 경로 선택 화면에서 상점 노드(🛒) 선택 |
| 전투 여부 | 없음 — 즉시 상점 오버레이 표시 |
| 배경 | 현재 지역의 전투 배경 이미지 유지 |
| 퇴장 | 나가기 버튼 또는 오버레이 외부 클릭 → 경로 선택 화면 복귀 |

## 2. 상점 오버레이 전체 레이아웃

### 2.1 레이어 구조

상점은 **전체 화면 오버레이** (z-index: 10, 모달 레벨)로 표시됩니다.

```
기존 레이어 (유지):
┌─ z:0   배경 이미지
├─ z:4   Zone A (상단 HUD) ← 유지 (소울 카운터 실시간 반영)
├─ z:4   Zone C (스킬 슬롯) ← 유지 (현재 장착 스킬 참조용)

상점 오버레이:
├─ z:10  어둡기 오버레이 (#000000 50%)
├─ z:11  상점 패널 (960 × 820px, 화면 중앙)
├─ z:12  상품 카드 호버/드래그
├─ z:13  확인 팝업 (구매/교체)
└─ z:14  상인 대사 버블
```

### 4.2 스킬 상품 카드 구성

```
┌──────────────────┐

│   데미지: 6        │  ← 핵심 수치, 12px, #FFF5E6 70%
│                    │
│ ─────────────── │  ← 구분선
│  ☀×2              │  ← 코스트, 아이콘 20×20 + 수치 16px Bold
└──────────────────┘
```

#### 가격 뱃지

| 속성 | 값 |
|:-----|:---|

| 아이콘 | ◆ 14×14px, `#D4A574` |
| 텍스트 | "◆12", 14px Bold, `#D4A574` |
| 부족 시 | 텍스트 색상 `#FF4444`, 배경 `#3A2020` |

#### 카드 프레임

| 속성 | 값 |
|:-----|:---|
| 배경 | `#1E1E24` → `#2A2A32` 그라데이션 |
| 테두리 | `#4A4A55` 2px, border-radius 8px |
| 나머지 구성 | 배틀씬 스킬 카드와 동일 (아이콘, 이름, 수치, 코스트 배치) |

### 4.3 스킬 카드 상태별 스타일

| 상태 | 불투명도 | 테두리 | 가격 뱃지 | 추가 효과 |
|:-----|:--------|:------|:---------|:---------|
| 구매 가능 | 100% | `#4A4A55` 2px | `#D4A574` 텍스트 | 없음 |
| 호버 | 100% | `#D4A574` 2px | `#D4A574` 텍스트 | y -8px 상승, `drop-shadow(0 4px 12px rgba(212,165,116,0.3))` |
| 소울 부족 | 60% | `#4A4A55` 1px | `#FF4444` 텍스트 | grayscale 40% |
| 판매 완료 | 30% | 없음 | — | 전체 grayscale + "SOLD" 오버레이 텍스트 (20px Bold, `#FF4444` 60%) |

### 4.4 상품 라인업 생성 규칙

| 속성 | 값 |
|:-----|:---|
| 스킬 구성 | 현재 클래스 전용 3장 + 범용 2장 |
| 가격 변동 | 기본가 ± 15% 랜덤 |
| 중복 방지 | 이미 장착 중인 스킬 제외 |
| 고정 시드 | 상점 노드 진입 시 확정 (나갔다 돌아와도 동일) |

## 9. 소울 경제 맥락

### 9.1 상점 방문 시점 예상 소울

| 시나리오 | 전투 경과 | 예상 소울 | 설명 |
|:---------|:---------|:---------|:-----|
| A (부유) | 일반 4 + 정예 1 | ~47 | 정예 승리 포함, 최대 소비력 |
| B (보통) | 일반 3 | ~24 | 평균적 진행 |
| C (빈곤) | 일반 2 | ~16 | 최소 진행, 하한선 |

> 소울 획득량 기준: 일반 8, 정예 15, 보스 25 (소울-경제-기획서 참조)

### 9.2 상품 가격 가이드라인

| 상품 | 가격 범위 | 시나리오 C(◆16)에서 | 시나리오 B(◆24)에서 | 근거 |
|:-----|:---------|:------------------|:------------------|:-----|
| 기본 스킬 | ◆8~12 | 1~2개 구매 가능 | 2~3개 | 최소 1개는 살 수 있어야 함 |
| 고급 스킬 | ◆15~20 | 0~1개 | 1~2개 | 시나리오 B에서 1개 여유 |
| 전리품 (일반) | ◆6~10 | 1~2개 | 2~3개 | 소모품 느낌 |
| 전리품 (희귀) | ◆12~18 | 0~1개 | 1~2개 | 투자 선택 |
| 스킬 슬롯 확장 | ◆25~35 | **불가** | 간신히 1개 | 장기 투자, 시나리오 A 전용 |

### 9.3 가격 설계 원칙

1. **시나리오 C 최소 보장**: 가장 싼 스킬(◆8) 최소 1개 구매 가능
2. **시나리오 B 선택의 여지**: 스킬 1~2개 또는 전리품 조합 가능
3. **시나리오 A 풀 투자**: 슬롯 확장 + 스킬 1개 또는 다량 구매 가능
4. **남은 소울 유도**: 항상 약간의 소울이 남도록 설계 → 다음 상점까지 축적 동기
