> 📋 개발 일정 및 제약 조건은 [마일스톤](./03.%20milestone.md) 참조

## 스코프 정의

### 반드시 포함 (Must Have)

#### 전직 시스템 (핵심 검증 대상)
- [ ] 기본 클래스 1개 (전사)
- [ ] 전직 1개 (팔라딘)
- [ ] 팔라딘 오라 1개 (지속 버프)
- [ ] 팔라딘 고유 카드 2-3장
- [ ] 전직 이벤트 UI (라운드 3 클리어 후)

#### 최소 전투 시스템 (전직 검증용)
- [ ] 에너지 시스템 (턴당 3 에너지)
- [ ] 기본 카드 5-7장 (공격, 방어, 스킬)
- [ ] HP 시스템
- [ ] 적 3종 (일반몬, 엘리트, 보스)

#### 최소 라운드 진행
- [ ] 전투 → 보상 카드 선택 → 다음 라운드 (5라운드)
- [ ] 라운드 구성: 일반몬 → 엘리트 → 전직 → 일반몬 → 보스

## 전직 시스템 상세 스코프

### 기본 클래스: 전사
**시작 덱 (7장)**
- 타격 x4 (1에너지, 6데미지)
- 방어 x2 (1에너지, 5방어)
- 강타 x1 (2에너지, 10데미지)

### 전직: 팔라딘
**컨셉**: 오라(지속 버프) + 고유 카드
- 팔라딘 오라 1개 (상세 설계 예정)
- 팔라딘 고유 카드 2-3장 (상세 설계 예정)

### 전직 조건
- 라운드 3 클리어 후 전직 이벤트 발생

## 성공 기준 (Definition of Done)

프로토타입이 성공했다고 볼 수 있는 조건:

1. **플레이 가능**: 3-5라운드 전투를 끝까지 플레이할 수 있음
2. **전직 체험**: 전직 선택 후 덱 플레이가 달라지는 것을 체감
3. **전략적 선택**: "어떤 전직을 고를까?" 고민이 생김
4. **재플레이 욕구**: 다른 전직으로 다시 해보고 싶은 마음이 듦
