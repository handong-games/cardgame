# 프로덕션 파이프라인

1인 개발 4주 프로토타입을 위한 **최소 생존 프로세스**

---

## 요약

| 영역 | 방식 | 시간 투자 |
|------|------|-----------|
| **버전 관리** | Trunk-Based (main + 태그) | 주 5분 |
| **빌드** | 수동 빌드 (주 1회) | 주 30분 |
| **테스트** | 수동 플레이테스트 | 주 2시간 |
| **에셋** | 텍스트 → AI 에셋 → 최종 | Week 3에 3시간 |
| **플레이테스트** | 자가 → 지인 → 온라인 | Week 3에 4시간 |

**총 오버헤드**: 주 3-4시간 (전체의 15-20%)

---

## 1. 버전 관리: Trunk-Based

```
main ─────●─────●─────●─────●
        v0.1  v0.2  v0.3  v0.4
        (W1)  (W2)  (W3)  (W4)
```

### 규칙
- `main` 브랜치 하나만 사용
- 주차별 Git 태그로 마일스톤 표시

### 커밋 메시지
```
[Week1] feat: 카드 데이터 구조 구현
[Week1] fix: 에너지 소모 버그 수정
[Week2] feat: 라운드 진행 시스템
```

### 태그
```bash
git tag week1-combat-loop
git tag week2-round-system
git tag week3-class-system
git tag week4-boss-integration
```

## 2. 빌드: 주 1회 수동

### 일정
- 매주 금요일 오후
- 빌드 후 직접 플레이테스트

### 폴더 구조
```
cardgame/
├─ builds/
│  ├─ week1/
│  ├─ week2/
│  ├─ week3/
│  └─ week4/
└─ .gitignore  # builds/ 폴더 제외
```

### 체크리스트
```markdown
## 빌드 전
- [ ] Unity 에러/경고 확인
- [ ] 플레이 모드에서 테스트
- [ ] 빌드 설정 확인 (해상도, 플랫폼)

## 빌드 후
- [ ] 실행 파일 정상 실행
- [ ] 마일스톤 목표 달성 확인
- [ ] builds/weekN/ 폴더에 저장
```

---

## 3. 테스트: 수동 플레이테스트

### 원칙
- 프로토타입 목표는 **"재미 검증"**, "버그 제로"가 아님
- 테스트 코드 작성 시간 → 기능 개발에 투자
- 자동화는 본개발 시작 시 고려

### 주간 테스트 시나리오

**Week 1**
- [ ] 카드 플레이 5회
- [ ] 에너지 0일 때 카드 플레이 시도
- [ ] 턴 종료 → 덱 리셔플 확인

**Week 2**
- [ ] 라운드 1-4 전체 플레이
- [ ] 카드 선택 후 덱에 추가 확인

**Week 3**
- [ ] 전직 이벤트 발생 확인
- [ ] 오라 발동 확인
- [ ] 팔라딘 카드 사용 가능

**Week 4**
- [ ] 5라운드 전체 플로우
- [ ] 보스전 클리어/패배
- [ ] 재시작 가능

## 4. 에셋: 3단계 전환

### Week 1-2: 텍스트 UI
```
[카드]
타격
1에너지 | 6데미지
```
- 기능 구현에만 집중
- 비주얼 작업 시간 0

### Week 3: AI 생성 에셋
| 용도 | 도구 |
|------|------|
| 카드 일러스트 | Midjourney, DALL-E |
| UI 요소 | Stable Diffusion |
| 아이콘 (보조) | Game-icons.net |
| 폰트 | Google Fonts |

- 시간: 2-3시간
- 스타일 일관성 유지

### Week 4+: 최종 에셋 (선택적)
- 프로토타입 검증 후 커스텀 에셋 고려

### 폴더 구조
```
Assets/
├─ _Placeholders/  # Week 1-2
├─ Art/
│  ├─ Icons/       # Week 3
│  └─ Cards/       # Week 4+
```

## 5. 플레이테스트: 점진적 확장

### 일정

| 주차 | 테스터 | 목적 |
|------|--------|------|
| Week 1-2 | 자가 | 기능 작동 확인 |
| Week 3 | 지인 2-3명 | 전직 시스템 피드백 |
| Week 4 | 자가 + 지인 | 전체 플로우 검증 |
| 이후 | 온라인 커뮤니티 | 확장 피드백 |

### Week 3 플레이테스트 질문지

```markdown
## 사전 정보
- 덱빌딩 게임 경험: [ ] 있음 [ ] 없음

## 플레이 관찰 (본인 체크)
- [ ] 카드 플레이 방법을 스스로 파악했는가?
- [ ] 에너지 시스템을 이해했는가?
- [ ] 카드 선택에서 고민했는가?

## 질문 (플레이어에게)
1. 어느 부분이 가장 재밌었나요?
2. 어느 부분이 가장 불편했나요?
3. 전직 후 플레이가 달라졌나요?
4. 다시 플레이하고 싶나요?
```

## 생존 팁

### 번아웃 방지
- **80% 룰**: 주간 목표 80% 달성 시 다음 주로
- **주말 휴식**: 금요일 빌드 후 토요일은 쉬기
- **과감한 컷**: Week 3에 "안 되겠다" 싶으면 피처 컷

### 시간 도둑 주의
- Asset Store 탐색: **1시간 제한**
- 아키텍처 고민: **2시간 이상 금지**
- 밸런싱: **"차이가 느껴지면" 성공**

### 동기 유지
- 주간 빌드 GIF 녹화
- Week 1 vs Week 4 비교 재생
- SNS에 진행상황 공유
