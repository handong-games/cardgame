# 밸런스 설계 가이드

이 문서는 스킬, 유물, 동전 시스템의 밸런스 설계 방향성을 제시합니다.

---

## 1. 핵심 설계 철학

### 1.1 목표: "뻔한 선택이 없는 게임"

**문제 정의**: 최적의 선택이 명확하면 플레이어는 고민 없이 해당 선택만 합니다.
- 예) 동전 뒷면이 가치가 없으면 → 앞면이 항상 좋다 (고민 없음)
- 예) 모든 상황에서 최강인 스킬 → 그것만 선택 (고민 없음)

**해결 방향**: 모든 선택에 상황에 따른 장단점을 부여합니다.
- 앞면/뒷면 모두 가치 있는 상황 만들기
- 스킬마다 빛나는 상황과 약한 상황 존재
- 유물 획득 시 이점과 단점 동시 고려

### 1.2 유리링의 6대 밸런스 원칙

| 원칙 | 설명 | 적용 예시 |
|------|------|----------|
| **등가 교환** | 리스크와 보상이 비례 | 높은 데미지 스킬 → 높은 코스트 또는 부작용 |
| **카운터 시스템** | 상성 관계로 다양성 유도 | 방어 관통 vs 고방어, 다수 공격 vs 단일 강공격 |
| **선택의 다양성** | 모든 선택지에 동등한 기대가치 | 1원 동전과 5원 동전의 트레이드오프 |
| **불확실성 관리** | 긴장감 + 통제 수단 | 동전 던지기 + 확률 조작 스킬 |
| **밸류 시스템** | 성능 수치화로 객관적 밸런스 | 데미지 10 = 블록 10 = 치유 7 등 |
| **지속적 성장** | 빌드 방향성에 따른 장기 목표 | 뒷면 특화, 안정형, 폭발형 빌드 |

## 2. 동전 시스템 밸런스 방향

### 2.1 앞면/뒷면의 가치 균형

**현재 문제**: 뒷면은 자원을 제공하지 않아 항상 앞면이 유리합니다.

**설계 방향**:
- 뒷면 전용 자원(섀도우 코인) 또는 뒷면 특화 스킬 도입
- 뒷면이 나올 때 발동하는 효과 설계
- 앞면/뒷면의 기대 가치를 동등하게 맞춤

**체크리스트**:
- [ ] 뒷면만으로 사용 가능한 스킬이 있는가?
- [ ] 뒷면 확률을 높이고 싶은 상황이 존재하는가?
- [ ] 앞면 특화와 뒷면 특화 빌드가 모두 유효한가?

### 2.2 동전 액면가 트레이드오프

**1원 동전의 장점**:
- 높은 안정성 (동전 3개 → 87.5% 확률로 1코스트 스킬 사용 가능)
- 저코스트 스킬과 시너지
- 세밀한 자원 관리

**5원 동전의 장점**:
- 높은 폭발력 (앞면 나오면 대형 스킬 즉시 사용)
- 고코스트 스킬과 시너지
- 올인 전략에 적합

**체크리스트**:
- [ ] 1원 선호 빌드와 5원 선호 빌드가 모두 존재하는가?
- [ ] 동전 교환(업그레이드/다운그레이드)에 의미 있는 선택이 있는가?

## 3. 스킬 밸런스 방향

### 3.1 밸류 기반 설계

모든 스킬 효과에 기준 밸류를 부여하여 객관적 비교를 가능하게 합니다.

**기준 밸류 개념** (구체적 수치는 별도 문서에서 정의):
- 데미지, 블록, 치유 등 기본 효과 간 교환 비율 정의
- 코스트별 적정 밸류 범위 설정
- 쿨다운, 조건부 발동 등에 따른 밸류 보정

**스킬 설계 시 확인사항**:
- [ ] 해당 스킬의 총 밸류가 코스트에 맞는가?
- [ ] 동일 코스트의 다른 스킬과 기대 가치가 비슷한가?

### 3.2 상황 의존적 효과

**원칙**: 모든 상황에서 최강인 스킬은 없어야 합니다.

**설계 패턴**:
| 패턴 | 설명 | 예시 |
|------|------|------|
| HP 조건부 | 플레이어 HP에 따라 효과 변동 | HP 50% 이하 시 데미지 2배 |
| 적 행동 조건부 | 적의 의도에 따라 효과 변동 | 적이 공격 예정일 때만 카운터 발동 |
| 콤보 의존 | 이전 스킬 사용에 따라 효과 증가 | 공격 스킬 후 사용 시 추가 데미지 |
| 자원 소모 | 추가 자원(HP, 동전)을 소모해 강화 | HP 10 소모하여 데미지 50% 증가 |

**체크리스트**:
- [ ] 이 스킬이 항상 최선인 상황이 있는가? (있다면 재설계 필요)
- [ ] 이 스킬이 빛나는 특정 상황이 있는가?
- [ ] 이 스킬이 약한 상황도 존재하는가?

### 3.3 시너지와 안티시너지

**시너지**: 특정 스킬/유물 조합 시 효과 증폭
- 뒷면 특화 스킬들끼리 시너지
- 저HP 빌드 스킬들끼리 시너지

**안티시너지**: 특정 조합이 효율을 낮춤
- 치유 스킬은 저HP 빌드와 상충
- 안정형 유물은 도박형 스킬과 상충

---

## 4. 유물 밸런스 방향

### 4.1 트레이드오프 필수

**원칙**: 모든 유물은 이점과 함께 단점(또는 기회비용)을 가집니다.

**설계 패턴**:
| 패턴 | 예시 |
|------|------|
| 직접적 단점 | 데미지 +20%, 받는 피해 +10% |
| 특정 빌드 제한 | 뒷면 확률 증가, 앞면 확률 감소 |
| 상황 제한 | 첫 턴에만 효과 발동 |
| 자원 소모 | 효과 발동 시 HP 감소 |

**체크리스트**:
- [ ] 이 유물을 "무조건 획득"하는 상황이 있는가? (있다면 재설계 필요)
- [ ] 이 유물을 획득하지 않는 것이 더 나은 빌드가 있는가?

### 4.2 빌드 아키타입 지원

유물은 특정 빌드 방향성을 강화하거나 약화시킵니다.

---

## 5. 빌드 아키타입

플레이어가 추구할 수 있는 대표적인 빌드 방향성입니다.

| 아키타입 | 핵심 전략 | 선호 요소 |
|----------|----------|----------|
| **안정형** | 꾸준한 자원 확보, 저코스트 스킬 활용 | 1원 동전, 저코스트 스킬, 확률 조작 |
| **폭발형** | 높은 리스크로 큰 보상 추구 | 5원 동전, 고코스트 스킬, 올인 유물 |
| **뒷면 특화형** | 뒷면 결과를 자원으로 전환 | 섀도우 코인 스킬, 뒷면 확률 증가 유물 |
| **저HP형** | 낮은 HP 유지로 조건부 보너스 극대화 | HP 조건부 스킬, 자해 유물 |
| **콤보형** | 스킬 연계로 효과 증폭 | 콤보 스킬, 연계 보너스 유물 |

---

## 6. 밸런스 검증 체크리스트

새로운 스킬/유물/시스템 추가 시 다음을 확인합니다.

### 등가 교환 검증
- [ ] 강력한 효과에 상응하는 비용/리스크가 있는가?
- [ ] 약한 효과에 보상(낮은 비용, 추가 효과)이 있는가?

### 선택의 다양성 검증
- [ ] 이 선택지를 "항상 선택"하는 상황이 없는가?
- [ ] 이 선택지를 "절대 선택 안 하는" 상황만 있지 않은가?
- [ ] 상황에 따라 최적 선택이 달라지는가?

### 빌드 영향 검증
- [ ] 특정 빌드와 시너지가 있는가?
- [ ] 특정 빌드와 안티시너지가 있는가?
- [ ] 어떤 빌드에서도 효율이 같지 않은가?

### 재미 요소 검증
- [ ] 플레이어가 이 요소를 사용할 때 고민하는가?
- [ ] 선택 결과에 따른 성취감/아쉬움이 있는가?
