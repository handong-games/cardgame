# 클래스 설계 가이드

이 문서는 코인 시스템 기반의 클래스 설계 철학과 각 클래스의 상세 사양을 정의합니다.

---

## 1. 코인 시스템과 클래스 철학

### 1.1 핵심 구조

```
앞면(Heads) = 정의(Justice) = 전사의 강점
뒷면(Tails) = 운명(Fate)   = 도적의 강점
지식(Knowledge)            = 마법사의 강점 (앞뒤 고루 활용)
탐욕(Greed) = 공용 시스템   = 보너스 토스 (리스크/리워드)
```

### 1.2 설계 원칙

1. **코인 결과와 클래스 정체성 연결**: 각 클래스는 특정 코인 결과에서 강점을 발휘
2. **대칭적 밸런스**: 전사(앞면)와 도적(뒷면)은 정반대 특성으로 균형
3. **중립적 선택지**: 마법사는 어느 결과든 활용 가능한 유연성 제공
4. **공용 시스템의 차별화**: 탐욕은 클래스별로 다른 시너지/안티시너지

---

## 2. 전사 (Warrior) - 정의의 수호자

### 2.1 기본 정보

| 항목 | 내용 |
|------|------|
| **상징** | 정의(Justice) |
| **코인 특화** | 앞면 |
| **철학** | 확실한 결과를 추구, 예측 가능한 안정성 |
| **플레이 스타일** | 안정적이고 꾸준한 데미지 딜러 |

### 2.2 강점

| 강점 | 설명 | 구현 방향 |
|------|------|----------|
| **안정적 데미지** | 앞면이 나오면 확실한 효과 발동 | 앞면 시 기본 데미지 +N |
| **앞면 확률 조작** | 앞면이 나올 확률을 높임 | "다음 토스 앞면 확률 +20%" 스킬 |
| **앞면 보너스** | 앞면이 나올 때 추가 효과 | "앞면 시 추가 방어 +3" 패시브 |

### 2.3 약점

| 약점 | 설명 | 의도 |
|------|------|------|
| **폭발력 부족** | 한 방에 큰 데미지 어려움 | 도적과의 차별화, 안정 vs 폭발 트레이드오프 |
| **뒷면 무용** | 뒷면이 나오면 효과 약함/없음 | 앞면 특화의 대가 |

### 2.4 스킬 설계 가이드

```
전사 스킬은 다음 패턴을 따름:
- 기본 효과: 앞면과 무관하게 발동하는 기본 데미지/방어
- 앞면 보너스: 앞면 시 추가 효과 (데미지 증가, 방어 증가 등)
- 뒷면 패널티: 없거나 약한 효과만 발동

예시:
  정의의 일격 (코인 2)
  - 기본: 데미지 8
  - 앞면 보너스: 추가 데미지 +6
```

## 3. 도적 (Rogue) - 운명의 추종자

### 3.1 기본 정보

| 항목 | 내용 |
|------|------|
| **상징** | 운명(Fate) |
| **코인 특화** | 뒷면 |
| **철학** | 불확실성을 수용하고 역이용, 터지면 강력 |
| **플레이 스타일** | 세팅 후 폭발적인 연속기 |

### 3.2 강점

| 강점 | 설명 | 구현 방향 |
|------|------|----------|
| **뒷면 보너스** | 뒷면이 나올 때 추가 효과 | "뒷면 시 추가 데미지 +8" |
| **뒷면 확률 조작** | 뒷면이 나올 확률을 높임 | "다음 토스 뒷면 확률 +20%" 스킬 |
| **저코스트 연속기** | 작은 코스트로 여러 번 공격 | 1코인 스킬 다수, 콤보 시스템 |

### 3.3 약점

| 약점 | 설명 | 의도 |
|------|------|------|
| **기본 데미지 낮음** | 폭발 없이는 약함 | 세팅의 중요성 강조 |
| **세팅 필요** | 강해지려면 준비(버프, 스택)가 필요 | 전사와의 차별화, 리스크/리워드 |

### 3.4 스킬 설계 가이드

```
도적 스킬은 다음 패턴을 따름:
- 기본 효과: 약한 기본 데미지
- 뒷면 보너스: 뒷면 시 강력한 추가 효과
- 연속기: 이전 스킬 사용 시 보너스

예시:
  운명의 단검 (코인 1)
  - 기본: 데미지 3
  - 뒷면 보너스: 추가 데미지 +5, 연속 사용 시 +2
```

## 4. 마법사 (Mage) - 지식의 탐구자

### 4.1 기본 정보

| 항목 | 내용 |
|------|------|
| **상징** | 지식(Knowledge) |
| **코인 특화** | 앞면/뒷면 고루 사용 |
| **철학** | 모든 결과를 이해하고 활용하는 지혜 |
| **플레이 스타일** | 상황 적응형, 원소 시너지 |

### 4.2 강점

| 강점 | 설명 | 구현 방향 |
|------|------|----------|
| **조건부 다양성** | 앞면/뒷면 각각 다른 효과 | "앞면: 화염 +3, 뒷면: 얼음 +3" |
| **원소 시너지** | 화염/얼음 등 원소 스택으로 효과 증폭 | 원소 스택 시스템 |

### 4.3 약점

| 약점 | 설명 | 의도 |
|------|------|------|
| **체력 낮음** | HP가 낮아서 생존력 약함 | 클래스 차별화, 유리대포 |
| **고코스트** | 강력한 스킬은 코인이 많이 필요 | 자원 관리의 중요성 |
| **특화 부족** | 앞면/뒷면 어느 쪽도 극대화 어려움 | 전문가 vs 만능의 트레이드오프 |

### 4.4 스킬 설계 가이드

```
마법사 스킬은 다음 패턴을 따름:
- 앞면 효과: 특정 원소 효과 (화염 데미지, 화상 등)
- 뒷면 효과: 다른 원소 효과 (얼음 데미지, 빙결 등)
- 원소 시너지: 같은 원소 스택 시 보너스

예시:
  원소 조화 (코인 2)
  - 앞면: 화염 데미지 6, 화염 스택 +1
  - 뒷면: 얼음 데미지 6, 얼음 스택 +1
  - 보너스: 화염+얼음 스택이 3 이상이면 추가 데미지 +5
```

## 5. 클래스 비교표

| | 전사 | 도적 | 마법사 |
|---|---|---|---|
| **상징** | 정의(Justice) | 운명(Fate) | 지식(Knowledge) |
| **코인** | 앞면 특화 | 뒷면 특화 | 앞뒤 고루 |
| **강점** | 안정, 확률 조작 | 폭발, 연속기 | 유연성, 원소 |
| **약점** | 폭발력 부족 | 세팅 필요 | HP/코스트 높음 |
| **플레이** | 예측 가능 | 터지면 강력 | 상황 적응 |
| **난이도** | 쉬움 | 중간 | 어려움 |

## 6. 탐욕 공용 시스템

### 6.1 개요

**탐욕 = 모든 클래스가 사용 가능한 공용 메커니즘**

- 탐욕 스택을 쌓아서 보너스 코인 토스 획득
- 보너스 토스에서 앞면이 나오면 코인 소모 (리스크)
- 보너스 토스에서 뒷면이 나오면 무료 획득 (보상)

### 6.2 클래스별 탐욕 시너지

| 클래스 | 탐욕 사용 시 | 평가 |
|--------|------------|------|
| 전사 | 앞면 = 손해 (강점이 리스크로) | **안티시너지** |
| 도적 | 뒷면 = 무료 (강점과 일치) | **시너지** |
| 마법사 | 양면 활용 (어떤 결과든 OK) | **중립** |

### 6.3 전략적 의미

- **전사**: 탐욕 사용을 피하거나, 뒷면 확률 유물로 보완 필요
- **도적**: 탐욕과 자연스럽게 시너지, 적극 활용 권장
- **마법사**: 상황에 따라 유연하게 선택

## 7. 밸런스 검증 체크리스트

### 클래스 정체성 검증
- [ ] 각 클래스가 특정 코인 결과에서 명확한 이점을 가지는가?
- [ ] 클래스 간 플레이 스타일이 충분히 차별화되는가?

### 강점/약점 검증
- [ ] 강점이 너무 압도적이지 않은가?
- [ ] 약점이 게임을 불가능하게 만들지 않는가?
- [ ] 강점과 약점이 균형을 이루는가?

### 탐욕 시너지 검증
- [ ] 탐욕이 특정 클래스에게만 유리하지 않은가?
- [ ] 탐욕 사용 여부가 전략적 선택이 되는가?
