# 스킬 생성 워크플로우 (Claude Code)

Claude Code와 함께 밸런스 있는 스킬을 생성하는 절차를 정의합니다.

---

## 1. 사전 준비

스킬 생성 전 다음 문서를 확인합니다.

### 필수 참조 문서
- [밸런스 설계 가이드](../design/balance-design.md) - 설계 원칙 확인
- [스킬 시스템](../02.%20proto/02.%20battle-scene/systems/skill-system/system.md) - 기술 구조 확인
- 기존 스킬 데이터 (`dev/proto/src/data/skills/`) - 기존 밸류 참고

### 확인 사항
- [ ] 추가하려는 스킬의 클래스(워리어, 팔라딘 등) 확인
- [ ] 해당 클래스의 기존 스킬 구성 파악
- [ ] 빌드 아키타입 중 어떤 방향을 지원할지 결정

---

## 2. 스킬 설계 단계

### 2.1 컨셉 정의

스킬의 역할과 특성을 먼저 정의합니다.

```
스킬명: [스킬 이름]
역할: [딜러/탱커/서포트 등]
빌드 방향: [안정형/폭발형/뒷면특화/저HP/콤보 등]
핵심 특징: [한 문장으로 설명]
```

### 2.2 효과 설계

| 항목 | 내용 |
|------|------|
| 기본 효과 | [데미지/블록/치유 등] |
| 조건부 효과 | [HP 조건, 적 행동 조건 등] |
| 부가 효과 | [버프/디버프/상태이상 등] |
| 코스트 | [동전 비용] |
| 쿨다운 | [턴 수] |

### 2.3 트레이드오프 설계

**질문 목록**:
- 이 스킬이 강한 상황은?
- 이 스킬이 약한 상황은?
- 이 스킬과 시너지 있는 요소는?
- 이 스킬과 안티시너지 있는 요소는?

---

## 3. 밸류 검증 단계

### 3.1 밸류 계산

스킬의 총 기대 밸류를 계산합니다.

**기본 공식** (예시):
```
총 밸류 = 기본 효과 밸류 + 조건부 효과 밸류 × 발동 확률 - 코스트 밸류 - 쿨다운 밸류
```

### 3.2 기존 스킬과 비교

동일/유사 코스트의 기존 스킬과 밸류를 비교합니다.

| 비교 항목 | 신규 스킬 | 기존 유사 스킬 |
|----------|----------|---------------|
| 코스트 | | |
| 기본 효과 밸류 | | |
| 총 기대 밸류 | | |
| 빌드 시너지 | | |

### 3.3 조정 기준

- **밸류가 높으면**: 코스트 증가, 쿨다운 추가, 조건 강화
- **밸류가 낮으면**: 효과 강화, 코스트 감소, 부가 효과 추가
- **특색이 없으면**: 조건부 효과 추가, 빌드 시너지 강화

## 4. Claude Code 프롬프트 템플릿

### 4.1 스킬 생성 요청

```
[클래스]에 새로운 스킬을 추가해주세요.

컨셉:
- 역할: [딜러/탱커 등]
- 빌드 방향: [안정형/폭발형 등]
- 핵심 특징: [설명]

요구사항:
- 밸런스 가이드 원칙 준수
- 기존 [클래스] 스킬들과 밸류 균형
- [특정 빌드]와 시너지

참조:
- docs/design/balance-design.md
- dev/proto/src/data/skills/[클래스].ts
```

### 4.2 밸런스 검증 요청

```
다음 스킬의 밸런스를 검증해주세요.

스킬 정보:
- 이름: [스킬명]
- 효과: [효과 설명]
- 코스트: [비용]

검증 요청:
1. 밸류 계산
2. 기존 스킬과 비교
3. 빌드 시너지/안티시너지 분석
4. 밸런스 조정 제안

참조:
- docs/design/balance-design.md
```

### 4.3 빌드 검증 요청

```
다음 빌드 조합의 밸런스를 검증해주세요.

빌드:
- 아키타입: [안정형/폭발형 등]
- 주요 스킬: [스킬 목록]
- 주요 유물: [유물 목록]

검증 요청:
1. 시너지/안티시너지 분석
2. 강점과 약점 식별
3. 다른 빌드 대비 경쟁력
```

## 5. 검증 체크리스트

스킬 추가 전 최종 확인 목록입니다.

### 등가 교환
- [ ] 강력한 효과에 상응하는 비용이 있는가?
- [ ] 코스트 대비 효과가 적절한가?

### 상황 의존성
- [ ] 이 스킬이 항상 최선인 상황이 없는가?
- [ ] 스킬이 빛나는 특정 상황이 있는가?
- [ ] 스킬이 약한 상황도 존재하는가?

### 빌드 호환성
- [ ] 특정 빌드와 시너지가 있는가?
- [ ] 안티시너지가 있는 빌드도 존재하는가?

### 기존 스킬과의 차별성
- [ ] 기존 스킬과 역할이 겹치지 않는가?
- [ ] 선택 시 고민할 요소가 있는가?

### 밸류 균형
- [ ] 동일 코스트 스킬들과 기대 밸류가 비슷한가?
- [ ] 극단적으로 강하거나 약하지 않은가?

---

## 6. 피드백 루프

### 6.1 테스트 시나리오

스킬 추가 후 다음 시나리오로 테스트합니다.

1. **단독 사용**: 스킬 단독으로 효율 확인
2. **시너지 조합**: 설계된 시너지 빌드에서 테스트
3. **안티시너지 조합**: 의도된 안티시너지 확인
4. **극단 상황**: 최상/최악의 상황에서 효과 확인

### 6.2 조정 기준

테스트 결과에 따른 조정:

| 결과 | 조정 방향 |
|------|----------|
| 항상 선택됨 | 단점 추가 또는 조건 강화 |
| 절대 선택 안 됨 | 효과 강화 또는 비용 감소 |
| 특정 빌드만 사용 | 의도한 것이면 OK, 아니면 범용성 조정 |
| 밸류 불균형 | 수치 조정 |
