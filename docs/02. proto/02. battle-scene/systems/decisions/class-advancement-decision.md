## 1. 문제 인식

### 로그라이크 반복 플레이의 지루함

**상황**: 전사 → 팔라딘 전직 과정이 처음에는 재미있지만, 죽고 다시 시작할 때 같은 과정을 반복하는 것이 지루해짐.

**핵심 문제**: 기존 시스템에서는 "팔라딘 카드"를 모아야 팔라딘이 되는데, 원하는 전직 카드가 보상에서 안 뜨면 리스타트하는 사태 발생.

```
[문제 시나리오]
1. 플레이어: "이번 런은 팔라딘 하고 싶다"
2. 카드 보상: [전사 카드] [마법사 카드] [도적 카드] ← 팔라딘 없음
3. 플레이어: "..." → 리스타트
4. 반복 → 지루함
```

**앞으로 더 심해질 문제**: 전직이 5개, 10개로 늘어나면 원하는 전직 카드가 뜰 확률이 더욱 낮아짐.

## 2. 해결 방향 탐색

### 2.1 초기 아이디어 목록

#### A. 카드 출현 조작 계열
- 전직 경로 선언 (목표 전직 선택 → 출현율 상승)
- 보장 슬롯 (보상 중 1장은 관심 전직 카드)
- 리롤 시스템 (보상 다시 굴리기)
- 카드 교환, 와일드카드

#### B. 전직 조건 변경 계열
- 선택제 전직 (카드와 무관하게 선택)
- 전직 포인트/퀘스트
- 시간/HP 기반 전직
- 적 처치 기반 전직

#### C. 시스템 변경 계열
- 전직 상점
- 전직 즉시 시작 (클리어한 전직으로 바로 시작)
- 승천 모드

### 2.2 1차 방향 결정: 전직 즉시 시작

처음에는 "한 번 클리어한 전직은 바로 그 상태로 시작"하는 방식을 고려.

**장점**: 구현 간단, 직관적
**한계**: 근본 문제(원하는 전직 카드가 안 뜸)를 해결하지 못함

### 2.3 근본 문제 재인식

> "전직 카드가 안 뜨면 리스타트"하는 상황 자체를 없애야 함

단순히 "전직 선택제"로 가면 다른 게임과 차별점이 없음.

## 3. 최종 방향 결정

### 3.1 핵심 아이디어

> **전직 전용 카드가 아닌 "공통 카드"로 전직 조건을 채운다**

기존: 팔라딘 카드 3장 → 팔라딘
변경: 수호 계열 공통 카드 3장 → 팔라딘

### 3.2 추가 아이디어

> **기본 클래스(전사/마법사/도적)에 따라 공통 카드풀이 다르다**

- 전사 공통 카드 → 전사 계열 전직들
- 마법사 공통 카드 → 마법사 계열 전직들
- 도적 공통 카드 → 도적 계열 전직들

### 3.3 결정적 아이디어

> **하나의 카드가 여러 전직에 기여할 수 있다**

```
예시:
- 팔라딘 인정 카드: A, B, C
- 버서커 인정 카드: A, D, E
- 검사 인정 카드: A, B, D, F

→ A 카드는 팔라딘, 버서커, 검사 모두에 기여!
→ 조건 동시 충족 시 플레이어가 선택
```

---

## 4. 이 선택의 이유

### 4.1 왜 공통 카드 기반인가?

| 이유 | 설명 |
|------|------|
| **리스타트 불필요** | 공통 카드는 항상 풀에 있으므로 "안 뜬다" 상황 없음 |
| **의도 반영** | 어떤 성향의 카드를 고르냐 = 어떤 전직으로 갈지 |
| **자연스러운 빌드** | "좋아서 고른 카드"가 전직으로 연결 |

### 4.2 왜 카드 공유 구조인가?

| 이유 | 설명 |
|------|------|
| **유연한 빌드** | 초반엔 열려있다가 점점 방향 확정 |
| **의미있는 선택** | "이 카드 고르면 팔라딘 확정인데..." 긴장감 |
| **후회 없음** | 어떤 카드든 최소 1개 전직에는 기여 |
| **선택권 부여** | 조건 동시 충족 시 플레이어가 선택 |

### 4.3 다른 게임과의 차별점

| 기존 게임 | 이 시스템 |
|----------|----------|
| 전직 카드 = 전직 고정 | 공통 카드 = 여러 전직 가능 |
| 카드 뜰 때까지 리스타트 | 어떤 카드든 전직에 기여 |
| 전직 1개 확정 | 조건 충족 시 선택권 |
| 단순 선택제 | 카드 선택 = 전직 방향 힌트 |

## 5. 고려했으나 채택하지 않은 방안

### 5.1 전직 경로 선언제

- 게임 시작 시 "나는 팔라딘 갈 거야" 선언 → 해당 카드 출현율 상승
- **미채택 이유**: 여전히 "특정 카드 뜨길 기다림" 요소 존재

### 5.2 순수 선택제

- 특정 라운드에서 전직을 그냥 선택
- **미채택 이유**: 차별점 없음, 카드와 전직의 연결고리 약화

### 5.3 전직 퀘스트제

- "방어 10회 사용" 같은 미션 달성 → 전직
- **미채택 이유**: 구현 복잡, 행동 제약 느낌

## 6. 향후 확장 가능성

- 하이브리드 전직: 두 전직 조건을 반반 채우면 새로운 전직
- 전직 성장: 인정 카드를 더 많이 모을수록 오라 강화
- 메타 프로그레션: 클리어한 전직은 시작 시 바로 선택 가능
