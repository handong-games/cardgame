## 작성 예시: 스킬 시스템

> 아래는 이 템플릿으로 작성한 예시입니다. 실제 작성 시 참고용으로 활용하세요.

### Part 1: 왜 만드는가

**기획 배경**

카드 드로우 시스템은 덱 관리의 복잡성이 높고 클래스별 차별화가 어려웠다. 전직 시스템 도입을 위해 스킬 기반 전투 시스템이 필요하며, 프로토타입 검증 기간을 확보하려면 지금 전환해야 한다.

**해결할 문제**

| 문제 | 현재 상황 | 해결 방향 |
|------|----------|----------|
| 복잡성 과다 | 덱/손패/버림패 관리 부담 | 모든 스킬 항상 접근 가능 |
| 차별화 부족 | 클래스별 플레이 스타일 동일 | 클래스별 고유 스킬 세트 |
| 확장성 한계 | 새 콘텐츠 추가 시 밸런스 난이도 높음 | 스킬 단위 독립적 밸런싱 |

**기대 효과 & 리스크**

| 구분 | 내용 | 대응 방안 |
|------|------|----------|
| 기대 효과 | 직관적 전투, 클래스 정체성 강화 | - |
| 리스크 | 선택지 과다로 결정 피로 | 스킬 슬롯 8개 제한 |

### Part 2: 무엇을 만드는가

**한 줄 정의**

> 스킬 시스템은 전투 행동 선택을 담당하며, 클래스별 고유 스킬과 코인 비용 구조를 통해 전략적 선택의 쾌감을 제공한다.

**감정 설계**

| 순간 | 목표 감정 | 설계 요소 |
|------|----------|----------|
| 스킬 선택 시 | 전략적 고민 | 상황별 최적 스킬이 다름 |
| 스킬 발동 시 | 타격의 쾌감 | 강렬한 시각/청각 피드백 |
| 새 스킬 획득 시 | 성장감, 기대감 | 빌드 방향에 영향 |
| 코인 부족 시 | 긴장, 아쉬움 | 자원 관리의 중요성 체감 |

**시스템 구조**

```
턴 시작 → 코인 토스(자원 획득) → 스킬 선택 → 효과 적용 → 턴 종료
                                    ↑
                           코인 비용 확인
                           쿨다운 확인
                           사용 횟수 확인
```

**핵심 규칙**

| 규칙 | 내용 | 예외 |
|------|------|------|
| 코인 비용 | 스킬 사용 시 앞면 코인 소비 (1-3개) | 0코인 패시브 스킬 |
| 턴당 사용 제한 | 스킬별 max_use 설정 | 무제한 스킬 존재 |
| 스킬 슬롯 | 최대 8개 장착 | - |

**주요 수치 기준**

| 항목 | 기준값 | 범위 | 비고 |
|------|--------|------|------|
| 1코인 스킬 데미지 | 6 | 5-8 | 기본 공격 수준 |
| 2코인 스킬 데미지 | 12 | 10-15 | 주력 스킬 |
| 3코인 스킬 데미지 | 18 | 15-22 | 궁극기급 |
